# Worked Examples

## Purpose

This file maps real workbook rows into the two structured layers:

- `project_date_fact`
- `derived_date_analysis`

Reference workbook:

- `FRM-INV-F0003 Inventus Master File_SD1 5G Tablet_Mar_18th_2026.xlsx`
- sheet: `Detailed Project schedule`
- reference analysis date: `2026-03-20`

These are example outputs for design validation.

They are not yet generated by a formal implemented skill.

## Example 10: Slack Guard Against False Positive

Reference workbook:

- `P9301A项目初步计划进度表1120（内部）.xlsx`
- sheet: `主计划`

Observed row:

- row `29`
- `parent_item = T0`
- `task = V1.0 驱动调试`
- `cycle = 25`
- `planned_start = 2026-02-03`
- `planned_finish = 2026-03-10`
- `note = 春节假期15天`

Interpretation:

- the raw cycle-vs-span check would look suspicious because the calendar span is longer than the stated cycle
- the note text provides a concrete holiday explanation
- anomaly confidence should be reduced rather than escalated

Guard effect:

- row `29`
- `task = V1.0 驱动调试`
- `affected_rule = cycle_vs_date_span_anomaly`
- `confidence_adjustment = down`
- reason:
  - the note explicitly states `春节假期15天`, which plausibly explains the longer calendar span
